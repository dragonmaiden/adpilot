# INPUT-CONTRACT.md

Use this contract to normalize data before analysis.

If fields are missing:
- list the important missing inputs
- state assumptions explicitly
- reduce confidence
- avoid false precision

## Minimum useful inputs

At minimum, try to obtain:
- date range
- spend
- revenue or purchase value
- purchases
- campaign-level or ad set-level breakdown

## Preferred business inputs

- average order value
- gross margin
- contribution margin
- target CPA
- target ROAS
- break-even CPA
- break-even ROAS
- growth priority
- promo notes
- cash or inventory constraints

## Preferred account inputs

- attribution model
- pixel / CAPI health
- structure notes
- objective mix
- retargeting presence

## Preferred performance inputs

- spend
- revenue
- purchases
- CPM
- CTR
- CPC
- LPV
- ATC
- checkout
- ROAS
- MER
- refunds

## Required behavior when data is incomplete

Before giving strong advice, ask:
- is the account profitable enough to scale?
- where is the bottleneck?
- is sample size sufficient?
- is structure harming performance?
- is creative likely the main issue?
- is tracking reliable enough?

If any answer is unclear, say so clearly.
